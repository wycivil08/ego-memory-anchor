# PRODUCT_V2.md — 忆锚 vs 大公司网盘：精细差异化设计

## 0. 核心命题

> 用户的替代方案不是 iCloud 或百度网盘——是"什么都不做，让记忆慢慢消散"。
> 忆锚要赢的不是存储之战，而是意义之战。

---

## 1. 专属档案（Memorial Profile）到底是什么

### 1.1 概念定义

专属档案不是"文件夹"。它是一个 **为逝者而存在的数字纪念空间**——
有名字、有面容、有日期、有关系、有家人共建、有文化节奏。

| 维度 | iCloud / 百度网盘 | 忆锚的专属档案 |
|------|------------------|-------------|
| 基本单元 | 文件夹 | **有身份的纪念空间**（姓名、头像、生卒日期、关系） |
| 组织方式 | 用户手动分类 | **自动时间线**（按 EXIF/手动日期生成生命叙事） |
| 权限模型 | "分享链接"或"共享文件夹" | **家庭角色制**（owner/editor/viewer，邀请制） |
| 情感属性 | 无 | **纪念日系统**（生日、忌日、清明，含农历） |
| 真实性保障 | 无 | **"原始记录"徽章**（不可移除，每条素材自动标注） |
| 叙事功能 | 无 | **家人注释**（多人视角，共同构建故事） |
| 文化适配 | 无 | **农历日历 + 清明提醒 + 文化节日整合** |
| 打开仪式感 | 打开一个工具 | **打开一个人的生命故事** |

### 1.2 专属档案的核心交互设计

**进入仪式**：用户从 Dashboard 点击一张逝者的卡片 → 进入的不是文件列表，
而是一条从上到下的 **生命时间线**——最早的记忆在底部，最近的在顶部。
页面顶端是逝者的头像、姓名、生卒日期，和家人写的一句话描述。

### 1.3 专属档案内的时间线设计

**时间线不是文件列表，是生命叙事。**

每条记忆卡片包含：
- 媒体类型图标 + 缩略图/内联播放
- 日期（精确度：日/月/年/不确定）
- SourceBadge: "📌 原始记录"
- 用户注释（可选）+ 注释者署名
- 贡献者头像

### 1.4 "原始记录"信任架构

这是忆锚与所有竞品/网盘的 **最深层差异化**。

**设计规则：**

1. 平台上的每一条素材都自动获得 **"原始记录"** 标记
2. 这个标记在 UI 上始终可见（每条卡片、详情页）
3. 标记不可被用户编辑或删除
4. 数据库中 `source_label` 字段 DEFAULT '原始记录'，无 UPDATE 入口（DB trigger 强制）
5. 如果未来引入任何 AI 辅助（如旧照片修复），修复后的版本 **必须** 标注 "AI 增强"，且原件永远保留

**心理学意义：** Julia Shaw 的研究警告 "AI 是完美的虚假记忆机器"。
我们的产品通过在视觉层面 **持续确认真实性**，为用户提供心理安全感。

### 1.5 纪念日系统

| 网盘 | 忆锚 |
|------|------|
| 无日历概念 | **自动识别生日、忌日** |
| 无文化适配 | **支持农历日期（清明、忌日常用农历）** |
| 无主动触达 | **温馨提醒："后天是爷爷的生日"** |
| 无仪式引导 | **提醒日附带一条随机记忆 + "点亮蜡烛"动效** |

### 1.6 家庭共建 — 增长引擎 + 心理学正确

| 网盘共享 | 忆锚家庭共建 |
|---------|------------|
| "我发给你一个文件" | "我们一起建设这个纪念空间" |
| 单向传递 | 多向贡献 |
| 无角色概念 | owner / editor / viewer |
| 无归属感 | "4 位家人共同守护" |

**邀请流程设计（极低摩擦）：**
1. Owner 点击 "邀请家人"
2. 选择角色：查看 / 编辑
3. 生成链接（无需填写邮箱 — 微信友好）
4. 复制链接 → 微信/短信发给家人
5. 家人点击链接 → 看到逝者信息 + "加入此记忆空间"
6. 注册/登录后自动关联

## 2. MVP 功能优先级 V2 调整

### 2.1 P0 优先级提升

| 功能 | V1 优先级 | V2 优先级 | 理由 |
|------|---------|---------|------|
| 家庭邀请 + 协作 | P1 → **P0** | P0 | 增长引擎，不是锦上添花 |
| 数据导出 | P1 → **P0** | P0 | 信任基石 — "导出"按钮放在醒目位置=信任 |
| Landing Page 隐私承诺 | 未列出 → **P0** | P0 | 回答"凭什么信你" |

### 2.2 完整 V2 P0 功能列表

```
P0（MVP 核心，全部必做）:
├── 用户认证（邮箱注册/登录）
├── 逝者专属档案 CRUD（含头像、生卒日期、关系、物种、描述）
├── 多档案支持
├── 5 种媒体上传（照片/视频/语音/文字/扫描件）
├── 批量上传 + EXIF 自动日期提取
├── 自动时间线视图（按年→月→日分组）
├── 原生媒体播放（照片/视频/语音内联播放）
├── 注释功能（每条素材 + 注释者署名）
├── "原始记录"徽章（自动、不可编辑）
├── 家庭邀请（链接制、角色制：owner/editor/viewer）   ← 提升
├── 数据导出（一键 ZIP 下载全部素材 + 元数据）        ← 提升
├── Landing Page（含隐私承诺区块 + 创始人故事区块）     ← 新增
├── 响应式设计（手机优先 + 桌面适配）
└── 空状态温暖设计
```

## 3. 差异化总结

> **iCloud 存的是文件。忆锚守护的是一个人的生命故事。**
